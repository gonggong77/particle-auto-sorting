# UPM 패키지화 및 1.0.0 초기 릴리즈

## 배경 / 동기

지금까지 `Particle Auto Sorting` Editor 도구는 두 Unity 프로젝트(`D:\particleSortingTest`, `D:\idleDante_test`)에 동일한 13개 .cs 사본이 복사되어 유지되고 있었다. 다른 Unity 프로젝트에 도구를 배포하려면 매번 `Assets/Editor/` 폴더를 수동 복사해야 했고, 두 곳의 사본이 시간이 지나면 어긋날 위험이 있었다. 이를 해결하기 위해 UPM(Unity Package Manager) Git URL 방식의 단일 패키지로 분리·공개했다.

## 작업 내용

### 1. 신규 패키지 레포 생성 (`D:\particleAutoSorting\`)

UPM 표준 폴더 구조로 셋업:

```
D:\particleAutoSorting\
├── package.json                  # name, version, displayName, unity, author, license, repository
├── README.md                     # 설치 방법(UPM UI / manifest.json), 사용법, 핵심 규칙
├── CHANGELOG.md                  # Keep a Changelog 1.0.0 항목
├── LICENSE                       # MIT
├── .gitignore                    # Unity 표준
└── Editor/
    ├── ParticleAutoSorting.Editor.asmdef    # 변경 없이 그대로
    ├── ParticleAutoSortingWindow.cs         (4개 partial)
    ├── ParticleAutoSortingWindow.List.cs
    ├── ParticleAutoSortingWindow.Table.cs
    ├── ParticleAutoSortingWindow.Bottom.cs
    ├── Analysis/                            (PrefabAnalyzer, BatchCounter)
    ├── Optimization/                        (SortingOptimizer)
    ├── Apply/                               (PrefabApplier, HierarchyReorderer)
    ├── Report/                              (CsvReportExporter)
    └── Data/                                (RendererInfo, PrefabData, MaterialGroupInfo)
```

소스는 `D:\particleSortingTest\Assets\Editor\` 의 13개 .cs + asmdef + .meta 파일을 디렉터리 구조 보존하며 그대로 복사 (코드 변경 0줄). 누락되어 있던 `Apply/HierarchyReorderer.cs.meta` 는 신규 생성 (GUID `127f79f633d84720ab004a2c902403e3`).

### 2. git 셋업 및 GitHub 푸시

```bash
cd /d/particleAutoSorting
git init
git branch -M main
git add .
git commit -m "feat: initial 1.0.0 release"
git remote add origin https://github.com/gonggong77/particleAutoSorting.git
git push -u origin main
git tag -a v1.0.0 -m "v1.0.0 — initial release"
git push origin v1.0.0
```

`gonggong77/particleAutoSorting` 레포는 사용자가 사전에 GitHub에서 빈 레포로 생성해둠. 38개 파일, 3066줄 추가로 첫 커밋. v1.0.0 태그 푸시 완료.

## 후속 작업 (사용자 영역, 선택)

기존 두 검증 프로젝트(`particleSortingTest`, `idleDante_test`)를 Embedded package로 전환하면 향후 코드 수정 시 단일 진실 공급원으로 운영 가능:

1. `Assets/Editor/` 내 ParticleAutoSorting 관련 .cs / asmdef / .meta 파일 삭제
2. `Packages/manifest.json` 에 git URL 또는 `file:D:/particleAutoSorting` (로컬 경로) 추가
3. Unity 재로드 → `Tools > particle auto sorting` 메뉴 노출 확인
4. 샘플 프리팹(`D:\particleSortingTest\Assets\samplePrefab\samplePrefab.prefab`)으로 분석/적용/CSV/Undo 회귀 검증

CLAUDE.md 분리 규칙(Claude는 Unity 프로젝트에 git 명령 안 함)에 따라 이 단계는 사용자가 직접 진행.
