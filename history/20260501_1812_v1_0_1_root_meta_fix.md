# v1.0.1 — 패키지 루트 .meta 누락 경고 수정

## 증상

UPM Git URL(`https://github.com/gonggong77/particleAutoSorting.git`)로 설치한 직후 Unity 콘솔에 다음 경고가 다수 출력:

```
Asset Packages/com.yurieiyi.particle-auto-sorting/CHANGELOG.md has no meta file, but it's in an immutable folder. The asset will be ignored.
Asset Packages/com.yurieiyi.particle-auto-sorting/Editor has no meta file, but it's in an immutable folder. The asset will be ignored.
Asset Packages/com.yurieiyi.particle-auto-sorting/LICENSE has no meta file, but it's in an immutable folder. The asset will be ignored.
Asset Packages/com.yurieiyi.particle-auto-sorting/package.json has no meta file, but it's in an immutable folder. The asset will be ignored.
Asset Packages/com.yurieiyi.particle-auto-sorting/README.md has no meta file, but it's in an immutable folder. The asset will be ignored.
```

**도구 자체는 정상 작동** (`Tools > particle auto sorting` 메뉴 노출, asmdef + .cs는 자체 .meta 보유). 단, 콘솔이 경고로 도배됨.

## 원인

UPM Git 패키지는 Unity가 **immutable(read-only)** 폴더로 마운트하므로, .meta가 없는 자산에 대해 Unity가 자동 생성을 못 함. 1.0.0 셋업 시 `Editor/` 하위 .cs와 asmdef는 원본 프로젝트(`D:\particleSortingTest\Assets\Editor\`)에서 .meta까지 함께 복사했지만, **패키지 루트의 5개 자산** (`package.json`, `README.md`, `CHANGELOG.md`, `LICENSE`, `Editor` 폴더 자체)은 원본에 존재하지 않던 신규 파일이라 .meta가 없었음.

`.gitignore`, `Documentation~/` 는 Unity가 import에서 제외하므로 .meta 불필요.

## 수정 내용

`D:\particleAutoSorting\` 루트에 5개 .meta 파일 신규 생성, 각각 안정 GUID 부여:

| 파일 | importer | GUID |
|---|---|---|
| `package.json.meta` | `PackageManifestImporter` | `588d17f446f94a2794f9a97ee0a5469b` |
| `README.md.meta` | `TextScriptImporter` | `4d3e18a339ca417da1530b11db1099f9` |
| `CHANGELOG.md.meta` | `TextScriptImporter` | `2422e79784f6441595d12772a7e77474` |
| `LICENSE.meta` | `DefaultImporter` | `e69412d1b95e4539af18c55aa1649ba7` |
| `Editor.meta` | `DefaultImporter` (`folderAsset: yes`) | `5f73f9d34be148d59391ccccf26a31a0` |

GUID는 한 번 정해진 뒤 절대 변경하지 않음 (Unity의 에셋 참조 안정성).

`package.json` 의 `version`을 `1.0.0` → `1.0.1` 로 bump, `CHANGELOG.md` 에 `[1.0.1]` Fixed 항목 추가.
